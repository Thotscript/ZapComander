Guia de Uso e Configuração

Aprenda a conectar sessões WhatsApp, criar bots inteligentes, configurar fluxos de conversa, usar a base de conhecimento RAG e agendar disparos automáticos.

O que é o ZapComander

O ZapComander é uma plataforma de automação WhatsApp com IA que permite criar bots personalizados que respondem mensagens, coletam dados de clientes, consultam APIs externas e enviam campanhas proativas agendadas — tudo vinculado ao seu número WhatsApp.

Keyword
Bot de Resposta
Ativado por palavras-chave. Consulta sua API e responde com IA.
RAG
Base de Conhecimento
Responde baseado em documentos que você carregou internamente.
Agendado
Bot Proativo
Dispara mensagens em horários definidos e continua a conversa.
Híbrido
Keyword + Agendado
Responde por keyword E dispara proativamente pelo cron.

Primeiros Passos

1
Crie sua conta
Acesse /login e cadastre-se com e-mail e senha. Sua conta isola seus bots e sessões dos outros usuários.
2
Autentique uma sessão WhatsApp
No painel, acesse Autenticar Sessões, informe o número e escaneie o QR Code com o WhatsApp do celular.
3
Crie um bot
Acesse Configurar Agentes e clique em + Novo Agente. Configure nome, palavras-chave, endpoint e prompt.
4
Teste o bot
Use o painel de Disparar Teste para enviar uma mensagem e verificar a resposta antes de ativar para clientes reais.
5
Ative e monitore
Com o agente ativo, ele interceptará automaticamente mensagens que contenham as palavras-chave configuradas.

Autenticar uma Sessão

Uma sessão é a conexão do seu número WhatsApp à plataforma. Cada sessão vincula um número a sua conta e permite que os bots enviados e recebidos por aquele número sejam gerenciados pela plataforma.

1
Informe o número
Digite o número no formato 55XXXXXXXXXXX (DDI + DDD + número, sem espaços ou símbolos).
2
Clique em Iniciar Sessão
O servidor iniciará um navegador Chromium headless que abrirá o WhatsApp Web e gerará o QR code.
3
Escaneie o QR Code
No WhatsApp do celular: Configurações → Dispositivos conectados → Conectar dispositivo. Aponte para o QR exibido na tela.
4
Aguarde a confirmação
O painel exibirá "Autenticado!" quando a conexão for estabelecida. A sessão persiste automaticamente — reinícios do servidor reconectam sem novo QR.
ℹ️
Persistência automática A sessão é salva localmente em tokens/. Ao reiniciar o servidor, sessões autenticadas são restauradas automaticamente em até 3 simultâneas sem necessidade de novo QR code.

Multi-Sessão

Você pode conectar múltiplos números WhatsApp à mesma conta. Cada sessão opera independentemente, com seus próprios bots vinculados via e-mail. Os bots criados na sua conta respondem em todas as sessões autenticadas associadas ao seu e-mail.

⚠️
Atenção: não use o mesmo número em dois navegadores simultâneos O WhatsApp Web permite apenas uma sessão ativa por número. Abrir o WhatsApp Web no navegador enquanto a sessão está ativa na plataforma causará desconexão.

Tipos de Bot

Bot de Resposta por Keyword

O tipo mais comum. O bot monitora todas as mensagens recebidas e é ativado quando a mensagem contém uma das palavras-chave configuradas. Consulta um endpoint externo (sua API) e usa a IA para formatar a resposta.

Quando usar: catálogos de produtos, listagem de imóveis, cardápios, consulta de status de pedidos.

Bot RAG (Base de Conhecimento)

Ao invés de consultar uma API externa, o bot pesquisa em documentos que você carregou internamente. Ideal quando as respostas são baseadas em textos fixos como manuais, FAQs e políticas.

Quando usar: suporte técnico, FAQ de produtos, políticas de empresa, documentação.

Bot Agendado (Proativo)

Dispara mensagens automaticamente para uma lista de contatos em horários definidos por expressão cron. Quando o contato responde, o bot entra no fluxo de conversa normal, coletando campos e consultando o endpoint.

Quando usar: cobrança, lembretes de consulta, promoções, follow-up de vendas.

Criar um Bot — Passo a Passo

1. Identidade do Bot

Campos
Nome do Agente obrigatório
Nome interno para identificar o bot no painel. Ex: Agente Imobiliário, Bot de Cobrança.
Descrição opcional
Texto curto exibido no card do bot. Ajuda a identificar sua função rapidamente.

2. Palavras-chave de Ativação

Define quais mensagens ativarão o bot. O sistema verifica se a mensagem do contato contém qualquer uma das palavras configuradas (não precisa ser exata).

Exemplos
imovel, casa, apartamento
Ativa quando a mensagem contém qualquer uma dessas palavras. Case-insensitive.
*
Asterisco sozinho: ativa para todas as mensagens recebidas. Use com cuidado — apenas um bot pode usar * de forma útil.
💡
Dica: Especificidade das keywords Se você tem múltiplos bots, use keywords específicas para cada um. Um bot imobiliário pode usar imovel, casa, alugar enquanto um bot de suporte usa problema, ajuda, erro, suporte. Evite sobreposição.

3. Endpoint de Dados

O endpoint é a URL da sua API que fornece os dados para o bot responder. O bot chama esse endpoint a cada mensagem e injeta a resposta no contexto da IA.

Configuração
URL do Endpoint obrigatório
URL completa da sua API. Ex: https://api.minhaempresa.com/imoveis
Método HTTP
GET, POST, PUT ou PATCH. Para enviar dados (como a mensagem do usuário), use POST.
Template da Requisição
JSON enviado no body para métodos POST/PUT. Use placeholders: {{message}} (mensagem do usuário), {{session}} (número da sessão), e qualquer campo coletado como {{nome}}, {{data}}. Deixe vazio para GET sem body.
Caminho da Resposta
Notação de ponto para extrair parte específica da resposta JSON. Se a API retorna {"data": {"imoveis": [...]}}, insira data.imoveis. Deixe vazio para usar toda a resposta.
Exemplo — Template de Requisição
{
  "query": "{{message}}",
  "session": "{{session}}",
  "data_inicio": "{{data_inicio}}",
  "limit": 5
}

4. Prompt da IA

O prompt instrui a IA (OpenAI GPT) sobre como formatar e apresentar a resposta ao usuário. Os dados retornados pelo endpoint são automaticamente anexados ao prompt antes do envio à IA.

💡
Como escrever um bom prompt Seja específico sobre o tom, formato e o que incluir. Indique que é para WhatsApp (respostas curtas, emojis se adequado). Mencione que os dados vêm de um JSON e peça para a IA usá-los.
Exemplo — Prompt para imobiliária
Você é um assistente de uma imobiliária. Com base no JSON de imóveis disponíveis,
responda de forma clara e amigável ao cliente no WhatsApp.
Destaque até 3 opções mais relevantes, incluindo preço, localização e
características principais. Seja conciso e use emojis adequados. 🏠

5. Controle de Conversa

Configurações de controle
Palavras de Encerramento
Palavras que encerram a conversa. Quando o usuário enviar qualquer uma, o bot finaliza e envia a mensagem de despedida. Ex: obrigado, tchau, encerrar, ok, parar
Mensagem de Encerramento
Texto enviado ao encerrar a conversa. Ex: Fico por aqui! Se precisar é só chamar. 😊
Máximo de Turnos padrão: 0 (ilimitado)
Número máximo de trocas de mensagem. Após atingir, o bot encerra automaticamente e envia a mensagem de despedida. 0 = sem limite.
Timeout de Inatividade padrão: 15 min
Minutos sem resposta do usuário para encerrar a sessão automaticamente. A sessão é limpa da memória após o timeout.
Limite de Resultados padrão: 0 (todos)
Quantos itens do array JSON da API enviar ao GPT. Use 1 para mostrar apenas o primeiro resultado. 0 = enviar todos.

6. Campos para Coletar

Defina dados que o bot precisa coletar do usuário antes de chamar o endpoint. O bot fará perguntas sequencialmente e só chamará a API após coletar todos os campos.

Formato JSON dos campos
[
  {
    "name": "nome",
    "question": "Qual é o seu nome?",
    "extractType": "uppercase"
  },
  {
    "name": "data_visita",
    "question": "Qual a data da visita? (DD/MM/AAAA)",
    "extractType": "date"
  },
  {
    "name": "quartos",
    "question": "Quantos quartos você precisa?",
    "extractType": "number"
  }
]

Cada campo coletado fica disponível como {{nome_do_campo}} no Request Template e no Self-Message Template.

Bot Agendado

O bot agendado envia mensagens automáticas para uma lista de contatos em horários definidos por expressão cron. Quando o contato responde, o bot entra no fluxo normal de conversa.

Configurações do Bot Agendado
Expressão Cron
Define o horário de disparo. Fuso horário: America/Sao_Paulo. Ex: 0 9 * * 1-5 = Seg–Sex às 9h.
Mensagem Automática
Mensagem enviada aos contatos no horário definido. Esta é a mensagem inicial — se o contato responder, a conversa continua.
Contatos
Números no formato WhatsApp: 55XXXXXXXXXXX@c.us. Separe por vírgula ou uma linha por contato. Ex: 5511999990001@c.us, 5511999990002@c.us
Template do Resumo (Self-Message)
Mensagem enviada ao número da própria sessão ao fim de cada conversa agendada. Útil para o operador receber um resumo estruturado da interação.

Expressões Cron — Referência

Formato: minuto hora dia-do-mês mês dia-da-semana

ExpressãoSignificado
0 9 * * 1-5Seg–Sex às 9h00
0 9 * * *Todo dia às 9h00
0 9,18 * * 1-5Seg–Sex às 9h e 18h
0 */2 * * *A cada 2 horas
*/30 * * * *A cada 30 minutos
0 10 1 * *Todo dia 1 do mês às 10h
0 8 * * 6Todo sábado às 8h
✅
Disparo imediato Use o botão ⏰ Disparar no card do bot para enviar a mensagem agora, independentemente do horário configurado no cron. Útil para testes e envios pontuais urgentes.

Base de Conhecimento (RAG)

O modo RAG (Retrieval-Augmented Generation) permite que o bot responda baseado em documentos que você carregou. Ao invés de consultar uma API, o sistema busca os trechos mais relevantes dos seus documentos e os injeta no prompt da IA.

1
Ative o modo RAG no bot
No formulário do bot, ative a chave Ativar modo RAG. Configure o número de trechos por consulta (padrão: 5) e a mensagem quando não há resultado.
2
Salve o bot
O botão RAG aparecerá no card do bot na listagem.
3
Selecione a sessão e carregue documentos
Clique em RAG no card, selecione a sessão WhatsApp e faça upload dos arquivos. Formatos aceitos: .txt, .md, .csv, .json, .html (máx. 2MB por arquivo).
4
Teste a consulta
Use o campo Testar Consulta para verificar quais trechos serão retornados para uma determinada pergunta.
ℹ️
Como funciona internamente Cada documento é dividido em pedaços de 1.200 caracteres com sobreposição de 200 caracteres. Cada pedaço é convertido em um vetor numérico (embedding) usando OpenAI. Na consulta, a pergunta também é vetorizada e comparada com todos os vetores armazenados — os mais similares (similaridade coseno ≥ 0.25) são usados como contexto.

Transcrição de Áudio

A plataforma transcreve automaticamente mensagens de áudio (PTT) recebidas. O texto transcrito é enviado de volta ao operador ou ao remetente, conforme configuração.

Pipeline de Transcrição

1
Áudio recebido
Mensagem de voz (PTT) ou áudio recebida na sessão.
2
Redução de ruído
FFmpeg aplica filtro afftdn para limpar o áudio antes da transcrição.
3
Transcrição Whisper local
Whisper local é executado com o modelo configurado (WHISPER_MODEL). Se falhar, usa a API OpenAI como fallback.
4
Envio do texto
O texto é enviado ao número da sessão (self) ou de volta ao remetente (sender), conforme TRANSCRIPTION_DESTINATION.

Testar um Bot

O Painel de Teste na seção de agentes permite enviar uma mensagem diretamente à primeira sessão ativa e ver a resposta antes de expor o bot a usuários reais.

⚠️
A resposta do teste é enviada ao número da própria sessão A mensagem de teste e a resposta da IA chegam no WhatsApp do número autenticado, não a um cliente externo.

Perguntas Frequentes

O bot não está respondendo — o que verificar?

1
Sessão está conectada?
Verifique em Minhas Sessões se o status é Conectado. Se não estiver, autentique novamente.
2
Bot está ativo?
No card do bot, verifique se o badge mostra ● Ativo. Clique em Ativar se estiver inativo.
3
Keyword correta?
Teste com a keyword exata configurada. Lembre que é case-insensitive mas a palavra precisa estar presente na mensagem.
4
Endpoint acessível?
Use o Disparar Teste para ver o erro retornado. Verifique se o endpoint está online e retornando JSON válido.

Posso ter múltiplos bots ativos ao mesmo tempo?

Sim. Cada bot monitora suas próprias keywords. Se uma mensagem contém keywords de múltiplos bots, apenas o primeiro match (por ordem de criação) é ativado. Use keywords específicas e não sobrepostas.

O que acontece se o contato não responder?

Após o Timeout de Inatividade (padrão: 15 minutos), a sessão de conversa é limpa da memória. Se o contato mandar uma mensagem depois, o bot tratará como uma nova conversa.

Posso usar o bot sem endpoint externo?

Sim, de duas formas: (1) ative o modo RAG e carregue documentos internos; (2) use um bot agendado simples sem campos obrigatórios — ele envia a mensagem configurada e aguarda resposta do contato sem precisar de API.

Como formatar a lista de contatos para o bot agendado?

Use o formato 55XXXXXXXXXXX@c.us onde 55 é o DDI do Brasil, seguido do DDD e número. Separe por vírgula ou uma linha por contato:

5511999990001@c.us
5521988887777@c.us
5531977776666@c.us

O bot responde mensagens de grupos?

Por padrão, o bot processa apenas mensagens diretas (chats 1:1). Mensagens de grupos são ignoradas pelo listener para evitar ativações indesejadas.
